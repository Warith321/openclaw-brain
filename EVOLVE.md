# EVOLVE.md - 自我进化系统

## 核心原则
1. **小步改动** — 每次只改一件事，改完验证
2. **git 兜底** — 所有改动立即 commit，随时可回滚
3. **量化评估** — 改动必须有可测量的效果
4. **通知老大** — 重大改动通过 QQ 通知

## 进化方向
- [ ] 记忆系统优化（减少检索失败）
- [ ] 技能库扩充（从 Clawhub 按需安装）
- [ ] SOUL.md 迭代（根据实际交互经验更新）
- [ ] 错误学习（记录失败案例，避免重复犯错）
- [ ] Moltbook 社区参与（向其他 agent 学习）

## 自我修复流程
1. 发现问题 → 记录到 `memory/errors.md`
2. 分析原因 → 制定修复方案
3. 小步实施 → git commit
4. 验证效果 → 更新版本记录
5. 通知老大 → QQ 汇报

## 回滚方法
```bash
git log --oneline          # 查看历史
git checkout <hash> -- .   # 回滚到某个版本
```
